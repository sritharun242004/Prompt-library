CSS Grid vs Flexbox: When to Use Each

August 14, 20243 min read
csswebfrontend

The standard answer — "use Grid for two-dimensional layouts, Flexbox for one-dimensional" — is technically accurate but practically useless. Most real layouts don't announce themselves as one- or two-dimensional. Here's the rule I actually use.

Use Flexbox when the content determines the layout. Use Grid when the layout determines the content.

That's the distinction that matters. Let me unpack it.

Flexbox: content-driven sizing

Flexbox items start from their natural size and then flex to fill (or shrink to fit) the available space. The layout adapts around the content.

The canonical Flexbox use case: a navigation bar where items have variable-width labels.

.nav {
  display: flex;
  gap: 1rem;
  align-items: center;
}

Each nav item takes its natural width. The container distributes the remaining space. If you add a new item, the others adjust. The content drives the layout.

Flexbox also excels at vertical centering (finally), equal-height columns in a card row, and distributing space between a left-aligned and right-aligned group:

.toolbar {
  display: flex;
  justify-content: space-between;
  align-items: center;
}

Grid: layout-driven placement

Grid starts from the tracks you define, then places items into them. The layout is the primary constraint; the content fills it.

The canonical Grid use case: a page layout with fixed regions.

.page {
  display: grid;
  grid-template-columns: 240px 1fr;
  grid-template-rows: auto 1fr auto;
  grid-template-areas:
    "header header"
    "sidebar main"
    "footer footer";
  min-height: 100vh;
}

You've defined the structure. Content goes where you tell it. Adding more content doesn't change the column widths — the layout holds.

Grid's minmax() and auto-fill/auto-fit make it ideal for card grids that adapt to the viewport:

.card-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(280px, 1fr));
  gap: 1.5rem;
}

This creates as many columns as fit at minimum 280px width, each growing to fill the row. No media queries needed.

The cases where the rule gets complicated

Wrapping rows: A row of tags that wraps to multiple lines looks one-dimensional but behaves two-dimensionally. Use Flexbox — the tags don't need to align across rows, and content-driven sizing is what you want (each tag the width of its text).

Subgrid: Sometimes you want card contents to align across cards — like a grid of cards where the title, body, and button should all share baselines across the row. That's Grid with subgrid:

.card-grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
}

.card {
  display: grid;
  grid-row: span 3;
  grid-template-rows: subgrid;  /* align internals to parent tracks */
}

Centering a single element: Either works. I default to Flexbox for the ergonomics:

.centered {
  display: flex;
  place-items: center;
  min-height: 100vh;
}

What I actually do in practice

I use Grid for page-level layouts and component regions (the card itself, a media-object, a dialog). I use Flexbox inside those regions — for the flex items within a card header, for a row of buttons, for anything where the items' intrinsic sizes should influence the distribution.

They compose well. A Grid item can be a Flex container. A Flex item can be a Grid container. The key is that you're choosing based on which axis of control matters: the outside-in structure of the layout, or the inside-out needs of the content.
