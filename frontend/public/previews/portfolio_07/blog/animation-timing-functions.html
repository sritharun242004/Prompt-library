Animation Timing Functions: The Science of Feel

Easing functions are the difference between animations that feel mechanical and ones that feel alive. This guide covers cubic-bezier curves, spring physics, and how to choose the right curve for every interaction.

March 5, 20258 min read

Every animation communicates something. A button that pops instantly says "digital." A button that springs back with a gentle overshoot says "physical." A loading bar that eases in and accelerates says "fast." The timing function is the difference between an animation that feels right and one that feels slightly off — even when users cannot articulate why.

Linear Is Almost Never Right

The CSS default timing function, ease, is already better than linear. But linear motion is genuinely wrong for almost every UI animation because nothing in the physical world moves at constant velocity. Objects accelerate and decelerate. They have momentum.

The only place I consistently reach for linear is progress bars and looping background animations — scenarios where you want uniform motion that does not start or stop.

The Four Named Curves

CSS provides ease, ease-in, ease-out, and ease-in-out. These cover most situations but lack nuance. ease is a good default — it decelerates at the end, which matches how objects come to rest. ease-out is ideal for elements entering the screen, because they start fast and settle smoothly. ease-in is for elements leaving the screen, because they start slow and accelerate away.

The problem with the named curves is that they are gentle. Real physical motion has more character than that.

Custom Cubic-Bezier Curves

The cubic-bezier function takes four numbers that define the control points of a Bezier curve. The curve maps time to progress — a steep early slope means fast initial motion, a gentle late slope means slow arrival.

My go-to curve for interactive feedback is cubic-bezier(0.34, 1.56, 0.64, 1). This curve overshoots slightly past the target and then settles back — it is the CSS approximation of a spring. It makes button presses, tooltip appearances, and dropdown reveals feel alive without requiring JavaScript.

For something entering the screen, I prefer cubic-bezier(0.22, 1, 0.36, 1) — a fast arrival that decelerates dramatically near the end. It suggests the element snapping into place.

Spring Physics vs. Easing Curves

Easing curves are approximations of physics. True spring physics — used by Framer Motion and React Spring — is computed frame by frame, which means the animation can be interrupted naturally. If a user dismisses a tooltip mid-animation, a spring will reverse from its current velocity rather than jumping back to the start.

For most hover effects and micro-interactions, a carefully chosen cubic-bezier curve is sufficient. For complex gesture-driven UIs, spring physics is worth the dependency.

Duration as a Design Decision

Duration is as important as the easing curve. A well-chosen curve with the wrong duration still feels wrong. The general rules: micro-interactions (hover, focus) should be 100-200ms. Component transitions (modals, drawers) should be 200-350ms. Page transitions can go up to 500ms if the motion is meaningful.

Anything over 500ms that is not a deliberate loading state will feel slow. Users are impatient. Animation should enhance the experience, not extend it.

Respecting User Preferences

Always respect the prefers-reduced-motion media query. Some users experience motion sickness or cognitive difficulty with animation. The CSS way is simple: wrap your transition properties in a check and either reduce the duration to near-zero or remove the animation entirely.

This is not just an accessibility checkbox. It is the right thing to do. The web should work for everyone, and motion is a choice, not a requirement.

Building Your Instinct

The fastest way to develop a feel for timing functions is to use a tool like cubic-bezier.com, create a curve, apply it to something real, and watch it. Pay attention to how it feels in context — not in isolation. A curve that looks perfect in a demo might feel sluggish on a small button, or frantic on a large modal.

Build a palette of three or four curves you trust and use them consistently across a project. Consistency is more important than perfection. A site where every animation uses the same timing vocabulary feels cohesive even if no individual animation is extraordinary.
