Understanding CSS Grid Layout

November 8, 20232 min read
  • css
  • layout

CSS Grid changed everything. Before Grid, we were hacking layouts together with floats and inline-blocks, then with Flexbox for one-dimensional layouts. Grid gives us true two-dimensional control over layout.

The Basics

You create a grid by setting display: grid on a container, then defining rows and columns.

.grid {
  display: grid;
  grid-template-columns: 200px 1fr 200px;
  grid-template-rows: auto 1fr auto;
  gap: 1rem;
}

The fr unit is Grid's superpower — it represents a fraction of the available space after fixed sizes are accounted for.

Placing Items

By default, Grid auto-places items left to right, top to bottom. But you can explicitly place items anywhere:

.sidebar {
  grid-column: 1 / 2;
  grid-row: 1 / 3;
}

.main {
  grid-column: 2 / 3;
  grid-row: 1 / 2;
}

Or use the shorthand:

.hero {
  grid-area: 1 / 1 / 2 / 4; /* row-start / col-start / row-end / col-end */
}

Named Template Areas

My favorite Grid feature — name your areas and place items by name:

.layout {
  display: grid;
  grid-template-areas:
    "header header header"
    "sidebar main main"
    "footer footer footer";
  grid-template-columns: 240px 1fr 1fr;
  grid-template-rows: auto 1fr auto;
}

.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main    { grid-area: main; }
.footer  { grid-area: footer; }

This is self-documenting CSS. You can see the layout visually in the property value.

Responsive Grids Without Media Queries

The auto-fill and auto-fit keywords let you create responsive grids without a single media query:

.cards {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(280px, 1fr));
  gap: 1.25rem;
}

This says: fill the row with as many columns as fit, but each column must be at least 280px wide. The grid adapts to the container width automatically.

The difference between auto-fill and auto-fit:

  • auto-fill creates empty tracks to fill the row
  • auto-fit collapses empty tracks, letting filled items grow

Grid vs Flexbox

Use Flexbox for one-dimensional layouts (a row of buttons, a nav bar). Use Grid for two-dimensional layouts (a card grid, a page layout).

In practice, you'll use both. A page might use Grid for the overall layout, with Flexbox used inside each component.

Alignment

Grid supports the same alignment properties as Flexbox:

.grid {
  justify-items: center;   /* horizontal alignment of items */
  align-items: center;     /* vertical alignment of items */
  justify-content: center; /* horizontal alignment of the grid */
  align-content: center;   /* vertical alignment of the grid */
}

Individual items can override with justify-self and align-self.

Summary

Grid is not a replacement for Flexbox — they complement each other. Grid excels at page-level layout and any time you need rows and columns to align. Once you understand the fr unit, named areas, and auto-fill/minmax, you have most of what you need to build any layout.
