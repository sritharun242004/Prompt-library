Vim Crash Course

February 10, 20232 min read

Vim is one of those tools that makes no sense until it suddenly does. The learning curve is real, but the productivity gains are worth it. Here's the crash course I wish I'd had.

The Modal Mental Model

Vim has modes. This is the big idea:

  • Normal mode — navigate and edit (default)
  • Insert mode — type text (i, a, o to enter; Esc to leave)
  • Visual mode — select text (v, V, Ctrl-v)
  • Command mode — run commands (:)

Think of Normal mode as your home base. You spend most time there.

Essential Normal Mode Commands

h j k l    — left, down, up, right
w b e      — word forward, back, end
0 $        — start/end of line
gg G       — top/bottom of file
Ctrl-d Ctrl-u  — half-page down/up

Operators + Motions

Vim commands are composable: operator + motion.

d  — delete
c  — change (delete + enter insert)
y  — yank (copy)
>  — indent

Examples:

dw     — delete word
d$     — delete to end of line
dd     — delete whole line
ciw    — change inner word (cursor anywhere in word)
yi"    — yank inside quotes
>ap    — indent a paragraph

The Dot Command

. repeats the last change. This is one of Vim's superpowers:

  1. ciw then type newWord — change a word
  2. Move to next occurrence
  3. . — repeat the change

Search and Replace

/pattern     — search forward
?pattern     — search backward
n N          — next/previous match
:%s/old/new/g  — replace all in file
:s/old/new/  — replace on current line

Splits and Tabs

:sp file   — horizontal split
:vsp file  — vertical split
Ctrl-w hjkl  — navigate splits
:tabnew    — new tab
gt gT      — next/previous tab

.vimrc Essentials

set number relativenumber
set tabstop=2 shiftwidth=2 expandtab
set ignorecase smartcase
set hlsearch incsearch
set clipboard=unnamedplus
let mapleader = ' '
nnoremap <leader>w :w<CR>
nnoremap <leader>q :q<CR>

Summary

  • Normal mode is home — always return to it
  • Learn operators (d, c, y) + motions (w, $, iw)
  • The dot command . multiplies your edits
  • :help <thing> is the best documentation you'll ever use
