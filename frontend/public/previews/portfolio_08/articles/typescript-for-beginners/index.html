TypeScript for Beginners

May 22, 20232 min read
  • typescript
  • javascript
  • fundamentals

TypeScript adds a static type system on top of JavaScript. You write TypeScript, it compiles to plain JavaScript. The benefit: your editor (and your colleagues) understand exactly what your code expects.

Basic Types

let name: string = 'Sanya'
let age: number = 30
let active: boolean = true
let scores: number[] = [95, 87, 100]
let pair: [string, number] = ['hello', 42]

Functions

function greet(name: string): string {
  return `Hello, ${name}!`
}

// Arrow with explicit return type
const add = (a: number, b: number): number => a + b

// Optional and default params
function createPost(title: string, draft = false, tags?: string[]) {
  // ...
}

Interfaces

interface User {
  id: number
  name: string
  email: string
  role: 'admin' | 'viewer'
  createdAt?: Date // optional
}

function displayUser(user: User) {
  console.log(`${user.name} (${user.role})`)
}

Type vs Interface

Use interface for objects you'll extend or implement. Use type for unions, intersections, and aliases:

type ID = string | number
type Status = 'idle' | 'loading' | 'success' | 'error'

type ApiResponse<T> = {
  data: T
  status: Status
  message?: string
}

Generics

Generics let you write reusable, type-safe code:

function first<T>(arr: T[]): T | undefined {
  return arr[0]
}

const firstUser = first<User>(users) // typed as User | undefined
const firstNum = first([1, 2, 3])    // inferred as number | undefined

Adding TypeScript to an Existing Project

npm install -D typescript @types/node
npx tsc --init

Start strict — but if the existing codebase is large, you can enable "strict": false and add strictness incrementally.

Rename files from .js to .ts one at a time. The compiler will tell you exactly what to fix.

Useful Utility Types

// Make all fields optional
type PartialUser = Partial<User>

// Make all fields required
type RequiredUser = Required<User>

// Only certain keys
type UserPreview = Pick<User, 'id' | 'name'>

// Exclude certain keys
type PublicUser = Omit<User, 'email'>

// Readonly
type ImmutableUser = Readonly<User>

Summary

  • TypeScript's type system catches bugs at compile time, not runtime
  • Start with interfaces for object shapes
  • Use generics for reusable utilities
  • Partial, Pick, Omit, and Readonly save a lot of boilerplate
  • Migrate gradually — you don't have to type everything on day one
