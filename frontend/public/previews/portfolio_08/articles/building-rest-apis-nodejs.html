Building REST APIs with Node.js

March 20, 20242 min read
  • node.js
  • api
  • javascript
  • backend

Node.js is one of the best choices for building REST APIs. It's fast, uses the same language as your frontend, and has a massive ecosystem. In this guide we'll build a complete REST API from scratch with Express.

Project Setup

Start by initialising a new project:

mkdir my-api && cd my-api
npm init -y
npm install express
npm install -D nodemon

Create src/server.js:

const express = require('express')
const app = express()
const PORT = process.env.PORT || 3000

app.use(express.json())

app.listen(PORT, () => {
  console.log(`Server running on port ${PORT}`)
})

REST Principles

A REST API maps HTTP verbs to CRUD operations:

| Method | Path | Action | |--------|------|--------| | GET | /users | List all users | | GET | /users/:id | Get one user | | POST | /users | Create a user | | PUT | /users/:id | Replace a user | | PATCH | /users/:id | Update fields | | DELETE | /users/:id | Delete a user |

Route Handlers

const router = express.Router()

// GET /users
router.get('/', async (req, res) => {
  const users = await User.findAll()
  res.json(users)
})

// GET /users/:id
router.get('/:id', async (req, res) => {
  const user = await User.findById(req.params.id)
  if (!user) return res.status(404).json({ error: 'Not found' })
  res.json(user)
})

// POST /users
router.post('/', async (req, res) => {
  const user = await User.create(req.body)
  res.status(201).json(user)
})

app.use('/users', router)

Middleware

Middleware functions run between the request and response. They're perfect for logging, authentication, and validation:

// Logger
app.use((req, res, next) => {
  console.log(`${req.method} ${req.url}`)
  next()
})

// Auth check
function requireAuth(req, res, next) {
  const token = req.headers.authorization?.split(' ')[1]
  if (!token) return res.status(401).json({ error: 'Unauthorized' })
  // verify token...
  next()
}

Error Handling

Define a global error handler as the last middleware:

app.use((err, req, res, next) => {
  console.error(err.stack)
  res.status(err.status || 500).json({
    error: err.message || 'Internal server error',
  })
})

Summary

  • Express gives you routing and middleware in a minimal package
  • Stick to REST conventions for predictable APIs
  • Always handle errors and return appropriate status codes
  • Use middleware for cross-cutting concerns like logging and auth
