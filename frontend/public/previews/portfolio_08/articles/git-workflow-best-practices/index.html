Git Workflow Best Practices

February 14, 20242 min read
  • git
  • workflow
  • tools

Git is the universal tool of software development, yet most developers only scratch the surface of what it can do. Here are the practices I've adopted after years of using Git daily.

Write Atomic Commits

Each commit should represent one logical change. Ask yourself: "If I had to revert this, would I want to undo everything in this commit?"

Bad: "Add user auth and fix the nav bug and update README"
Good:

feat: add JWT authentication middleware
fix: correct active link highlight in nav
docs: update README with auth setup instructions

Conventional Commits

The Conventional Commits spec gives commit messages a machine-readable structure:

<type>[optional scope]: <description>

feat(auth): add password reset flow
fix(api): handle null user in /profile endpoint
docs: add contributing guide
chore: upgrade eslint to v9

Types: feat, fix, docs, style, refactor, test, chore, perf, ci.

Branching Strategy

For most projects, a simple flow works well:

  • main — production-ready code only
  • develop — integration branch
  • feature/xxx — one feature, merge into develop
  • fix/xxx — one bug fix
  • release/x.x.x — release preparation
git checkout -b feature/user-avatar
# ... work ...
git push origin feature/user-avatar
# open a pull request

Interactive Rebase

Before merging, clean up your commit history:

git rebase -i main

This opens an editor where you can pick, squash, reword, or drop commits. Use it to turn a messy WIP history into a clean, reviewable set of changes.

Useful Aliases

[alias]
  st = status -sb
  lg = log --oneline --graph --decorate --all
  undo = reset HEAD~1 --mixed
  aliases = config --get-regexp ^alias\\.

Git Hooks

Automate quality checks with hooks. Put this in .git/hooks/pre-commit:

#!/bin/sh
npm run lint && npm test

Or use Husky for a project-wide solution:

npx husky init
echo "npm test" > .husky/pre-commit

Summary

  • Small, atomic commits make history readable and reverting safe
  • Conventional Commits add structure and enable automation
  • Interactive rebase keeps your branch history clean before merging
  • Hooks enforce standards automatically
