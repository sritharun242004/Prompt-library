Getting Started with Linux

June 20, 20222 min read
  • linux
  • tools
  • workflow

Linux is the operating system of servers, containers, and serious developers. Even if you use macOS or Windows daily, knowing Linux will make you significantly more capable. This is the guide I wish I'd had.

Why Linux?

  • All servers run Linux (or BSD)
  • Understanding it helps with Docker, CI/CD, cloud platforms
  • The command line is unmatched for automation
  • It's free and open source — you can read and modify everything

The Filesystem

Linux has one tree, starting at /:

/           root of everything
├── bin/    essential binaries (ls, cp, etc.)
├── etc/    configuration files
├── home/   user home directories
│   └── tania/
├── tmp/    temporary files (cleared on reboot)
├── usr/    user programs and libraries
└── var/    logs, databases, spool files

Your home directory is ~ — a shortcut for /home/yourusername.

Essential Commands

# Navigation
pwd             # where am I?
ls -la          # list with hidden files, permissions
cd ~/projects   # change directory
cd ..           # go up one level

# Files
cp file dest    # copy
mv file dest    # move/rename
rm file         # delete (permanent!)
rm -rf dir/     # delete directory recursively
mkdir -p a/b/c  # create nested dirs

# Viewing
cat file        # print file
less file       # paginate (q to quit)
head -20 file   # first 20 lines
tail -f log     # follow a log file live

# Search
grep -r "pattern" .   # search recursively
find . -name "*.ts"   # find files by name

Package Management

Different distros have different package managers:

# Debian/Ubuntu
sudo apt update && sudo apt install nodejs

# Arch Linux
sudo pacman -Syu nodejs

# Fedora
sudo dnf install nodejs

File Permissions

-rw-r--r-- 1 tania tania 1024 Jan 01 file.txt

Permissions: owner / group / everyone. Three bits each: read (4), write (2), execute (1).

chmod 755 script.sh   # rwxr-xr-x
chmod +x script.sh    # add execute for all
chown tania file.txt  # change owner

Process Management

ps aux              # all running processes
top / htop          # interactive monitor
kill -9 PID         # force kill process
Ctrl-C              # stop foreground process
Ctrl-Z              # suspend (bg to resume in background)

SSH

ssh user@server.com        # connect
ssh-keygen -t ed25519      # generate key
ssh-copy-id user@server    # install public key

Once your key is installed, you log in without a password.

Summary

  • The filesystem starts at /; home is ~
  • grep, find, less, tail -f are your daily tools
  • Permissions are owner/group/world; use chmod and chown
  • SSH with key authentication — never use passwords on servers
