Why I still like plain JavaScript

July 22, 2023
  • javascript
  • opinion
  • frontend

Every few months there's a new JavaScript framework that promises to solve all your problems. I've used most of them. And I still think there's something valuable about writing plain JavaScript.

Not because frameworks are bad — React and Svelte and SolidJS are genuinely excellent tools. But because:

You understand what's happening. When something breaks in plain JS, you can trace it. When something breaks in a framework, you're often debugging the framework, not your code.

It forces you to learn the platform. The DOM API, fetch, IntersectionObserver, ResizeObserver, custom events — these are surprisingly capable. Most "why do I need a library for this?" moments come from not knowing they exist.

Small scripts stay small. Not every interactive element on a page needs to be a React component. A 20-line script that handles a dropdown doesn't need a build step.

I write React daily. I also keep a handful of small vanilla JS snippets for the things that don't need it. The right tool for the job and all that.
