Thoughts on Open Source

August 19, 2023
  • open-source
  • career

I've been maintaining open-source projects for a few years now. Here are some honest thoughts.

The Good Parts

It's made me a better developer. When code is public, you care about it differently. The naming matters. The API matters. The documentation matters. The pressure is real but useful.

The connections are real too. Some of the most interesting conversations I've had started with a GitHub issue.

The Exhausting Parts

Feature requests never stop. Some are thoughtful and well-reasoned; many are "can you just add X?" where X would take a week and serve a use case I don't personally have.

The hardest thing is saying no. Every "won't fix" feels like a failure even when it's the right decision.

The Thing About Stars

GitHub stars are not a measure of impact. They're a measure of discoverability and a certain kind of novelty. Some of my most useful work has barely any stars. Some projects I'm proud of barely get used.

Why I Keep Doing It

The writing I do about the projects has led to more work opportunities than the projects themselves. The blog posts explaining decisions, the guides showing how to use the tool — these are visible in a way that code rarely is.

And occasionally someone tells me that something I built saved them hours of work. Those messages are why.
