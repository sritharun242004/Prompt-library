TIL about CSS :has()

May 3, 2024
  • css
  • til

Today I finally sat down and properly understood CSS :has(). I'd seen it mentioned but kept ignoring it. That was a mistake.

:has() is sometimes called the "parent selector" but it's more accurately a relational selector. It lets you style an element based on what it contains.

/* Style a form that contains an invalid input */
form:has(input:invalid) {
  border-color: red;
}

/* Style a card that contains an image */
.card:has(img) {
  display: grid;
  grid-template-columns: 200px 1fr;
}

/* Style a label that's next to a checked checkbox */
label:has(+ input:checked) {
  font-weight: bold;
}

The last one is wild. You can now style things based on the state of sibling elements — without JavaScript.

Browser support is now excellent (Chrome 105+, Safari 15.4+, Firefox 121+). Time to start using it.
