Command Palette Patterns That Actually Work

A comprehensive look at the interaction models, shortcut hierarchies, and search algorithms behind the best command interfaces in modern software.

Interaction DesignDeveloper ToolsPatterns

The command palette is one of the most consequential interaction patterns to emerge in the last decade of software design. What started as a power-user shortcut in text editors has become the primary navigation surface in tools like Linear, Raycast, Vercel, and VS Code. Done well, it collapses the entire product into a single keystroke.

The first design decision that separates great command palettes from mediocre ones is scope. Narrow command palettes (only global actions) feel fast but hit a ceiling. Contextual command palettes (actions + current-page results + recent items) are dramatically more useful but require careful result ranking. The ranking problem is genuinely hard: you need to balance recency, frequency, semantic relevance, and role-based permissions all at once.

Keyboard shortcut design within a command palette is its own discipline. The temptation is to expose every feature behind a searchable label and be done with it. But the best palettes also surface shortcut hints inline, teach the user keyboard shortcuts passively, and — critically — respect the user's existing muscle memory from other tools. Fighting muscle memory is a losing battle.

The most underbuilt part of command palettes is the empty state and the zero-results state. Both need to be curated: zero-results should not be a dead end but a suggestion machine, pointing the user toward alternate search terms or related actions. The best zero-results UX I've seen is Raycast's — it surfaces Quicklinks, extensions, and a "Search on the web" escape hatch simultaneously.
