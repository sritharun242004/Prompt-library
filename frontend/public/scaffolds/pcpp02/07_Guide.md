# 07 — Guide
## Engineering Standards · pcpp_platform_02

### 1. CreatorGrid Checklist
- **Navigation:** Is the sidebar fixed and accessible on all screen sizes?
- **Hierarchy:** Are categories organized by "Technique" rather than generic "Work"?
- **Typography:** Are spec tables using the Monospace font (JetBrains Mono)?
- **Borders:** Are all dividers using the 1px hair-line (#EEEEEE) style?

### 2. Coding Conventions
- **CMS:** Strictly use `Contentlayer` for all blog, technique, and museum data to keep the build fast and type-safe.
- **State:** Use Zustand for managing the global `sidebarOpen` and `activeCategory` states.
- **Images:** Use `loading="lazy"` for all grid items except the first 4 to ensure instant page loads.
- **Monetization:** All digital assets must be served via secure, time-limited links generated by the payment provider (Stripe/Lemonsqueezy).

### 3. File Structure
```
src/
  app/
    (marketplace)/  # Homepage, Shop, Category pages
    photography/    # Dynamic technique-based routing
    museum/         # Lens Museum archive
    blog/           # Tutorials and gear reviews
  components/
    layout/         # Sidebar, SidebarLink, MobileHeader
    photography/    # GallerySwitcher, ProjectCard, TechniqueHeader
    museum/         # LensCard, SpecTable, SampleGallery
    shop/           # AssetCard, PriceTag, CheckoutOverlay
  store/            # useNavStore, useCartStore
  lib/              # contentlayer-utils, payment-sdk
```

### 4. Definition of Done
- Nested sidebar navigation works with zero layout shift.
- Dynamic technique pages correctly filter and display projects from the CMS.
- Lens Museum spec tables are fully responsive and legible on mobile.
- Shop checkout is integrated and digital assets are delivered via automated webhooks.
