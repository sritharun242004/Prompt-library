## Base Prompt

### Section 1 — Role

You are a senior product designer and motion architect specializing in high-fidelity cinematic landing pages, motion-driven scrollytelling, and premium hardware identities. You understand that for world-class products, the website is an "Experience." You master the "Scroll-to-Discover UI," where image-sequence scrubbing (via Canvas/GSAP), pinned narratives, and massive typography are used to create a "Digital Cinema" feel. You reject "standard" layouts in favor of the "Apple" philosophy: full-bleed interactive chapters, fluid motion transitions, and a focus on "Story before Specs." You design for "Emotional Impact," ensuring the path from a breathtaking hero animation to a dense technical specification grid is seamless, professional, and profoundly immersive.

---

### Section 2 — Application Overview

This is a premium digital flagship for a high-end consumer technology product (e.g., AirPods). The audience consists of design-conscious consumers and audiophiles seeking "The Magic of Innovation." The goal is to provide a "Visual Journey" that uses scroll-triggered motion to deconstruct the product's features, followed by a transition into high-density technical details and a sticky buying interface.

The application covers four primary views: Cinematic Hero (Scrollytelling), Feature Deep-Dive (Pinned), Technical Spec Grid, and Sticky "Buy Now" Utility. The primary goal is a "Pre-order/Buy" action. Secondary goal is "Feature Discovery."

---

### Section 3 — Brand Voice & Mood

The feeling is awe-inspiring, technically precise, and magically responsive. Like watching a premium cinematic trailer where you control the speed with your thumb. The brand is authoritative and evocative — it focuses on "Innovation you can Feel."

Copy is evocative, punchy, and confident. Headlines focus on "The Sensation" — "Magic like you've never heard," "Rebuilt from the sound up," "Pro-level performance." Sentences are short. No marketing jargon. No exclamation marks.

The visual mood: "Emotional Tech Minimalism." High-fidelity product renders on pure white or black foundations. Expansive whitespace (or black space). Visual chunking — one section, one story.

Vibe word: IMMERSIVE.

---

### Section 4 — Core Features & Functionality

1. **Image-Sequence Scrubbing Engine** — a core component using `<canvas>` and GSAP ScrollTrigger to play a high-fidelity image sequence (0-120 frames) forward and backward as the user scrolls.
2. **"Pinned" Narrative Chapters** — a layout system that "locks" the viewport while text overlays fade and translate (opacity 0 → 1, y: 20 → 0) to tell specific feature stories.
3. **Interactive "Hotspot" Callouts** — subtle clickable points on the product deconstruction that reveal deep-dive technical pop-ups without breaking the immersive scroll flow.
4. **Adaptive Sub-Navigation** — a sticky secondary bar that tracks the product name and current chapter, allowing users to jump between "Audio," "Battery," and "Specs."
5. **High-Density Technical Spec Grid** — a transition at the end of the visual story into a clean, categorized table of technical details (Size, Weight, Battery, Connectivity).

---

### Section 5 — Design Specifications

**Visual style:** Emotional Tech Minimalism. High-fidelity renders. Full-bleed layouts. Strictly 1px borders for technical sections. Zero decorative "fluff" — the product geometry is the visual.

**Color mode:** Strictly High-Contrast (Pure White or Pure Black backgrounds).

**Color palette:**
- Background: `#FFFFFF` (Pure White — the storytelling foundation)
- Surface / Specs: `#F9FAFB` (Subtle grey for comparison cards)
- Border / Line: `#D2D2D7` (Zinc-300 ghostly separators)
- Primary text: `#1D1D1F` (Apple Ink near-black)
- Muted text: `#6E6E73` (Neutral grey for metadata and captions)
- Accent (Interact): `#0071E3` (Apple Blue — brand signature)
- Accent Success: `#00D26A` (Laser Green — for "In Stock" indicators)

**Total colors: 7 values, 4 functional categories. No additional colors.**

**Typography:** SF Pro throughout (or Inter as high-fidelity fallback).
- Display / Hero heading: `clamp(48px, 8vw, 80px)`, weight 700, letter-spacing `-0.04em`, line-height 1.05
- Section heading (H2): `clamp(24px, 4vw, 36px)`, weight 600, letter-spacing `-0.01em`, line-height 1.2
- Narrative Body: `21px`, weight 400, letter-spacing `0`, line-height 1.4
- Technical / Spec: `12px`, weight 600, letter-spacing `0.05em`, line-height 1.2 (uppercase)
- Max line length body text: 50 characters (centered)

**Spacing:** 20pt base unit.
- Section scroll length: `200vh` to `500vh` for scrollytelling sequences.
- Container max-width: `1024px` for text; `1440px` for spec grids.
- Vertical padding: `120px` standard.

**Border radius:**
- Buttons: `100px` (pill-shaped)
- Feature cards: `18px` (soft rounded)
- Technical grids: `0px` (sharp hair-lines)
- Inputs: `12px`

**Responsive:** Mobile-first scrollytelling. Image sequences must be optimized for mobile bandwidth (WebP). Pinned text overlays stack vertically above or below the product center on small screens.

**Accessibility:** WCAG AA. Motion-respecting transitions (`prefers-reduced-motion`). All canvas sequences must have descriptive fallback DOM text.

**Motion:**
- Scrub Logic: 1:1 mapping of scroll position to frame index.
- Ease Transitions: `Power2.easeOut` for text reveals.
- Fade Slide: `opacity 0 → 1`, `y: 20 → 0`, `400ms`.
- Forbidden: bounce easing, parallax, infinite scroll loops.
- Respect: `prefers-reduced-motion: reduce` → disable all animations and transitions.

### Section 7 — Technical Specifications

- **Framework:** Next.js 14 App Router, React 18, TypeScript strict mode.
- **Animation Engine:** GSAP (GreenSock) + ScrollTrigger plugin.
- **Rendering:** HTML5 `<canvas>` (2D Context) for performant sequence scrubbing.
- **Optimization:** Pre-loading hook for 120+ frame array (WebP).
- **Styling:** Tailwind CSS v3 with custom "Emotional Tech" tokens.
- **State:** Zustand for managing `activeChapter` and `preloadingProgress`.
- **Performance:** Sub-1.5s LCP (Initial frame). 60FPS scrubbing performance.

**State Snippet (store/scroll.ts):**
```typescript
interface ScrollStore {
  currentFrame: number;
  activeChapter: 'intro' | 'audio' | 'specs';
  isPreloaded: boolean;
  setFrame: (index: number) => void;
  setChapter: (id: string) => void;
}
```
