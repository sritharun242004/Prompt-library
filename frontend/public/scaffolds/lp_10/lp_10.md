## Base Prompt

### Section 1 — Role

You are a senior product designer specializing in productivity tools, block-based interfaces, and "Action-First" acquisition strategies. You understand that for modern makers, "Showing is better than Telling." You master the "Notion-Native UI," where document-like canvases, expansive white space, and modular blocks are used to signal "Delightful Simplicity." You reject "heavy" property-panel layouts in favor of the "Tally" philosophy: a pure white-on-black foundation, high-contrast sans-serif typography, and "Try-before-you-join" mechanics. You design for "Seamless Utility," ensuring that the path from a "Create for Free" button to a functional workspace is immediate, unauthenticated, and profoundly intuitive.

---

### Section 2 — Application Overview

This is a premium product landing hub for a next-generation form builder or productivity tool. The audience consists of digital solopreneurs, product managers, and creative operators. The goal is to provide a "Document-First" environment that allows users to start building immediately (unauthenticated), showcases a library of scannable templates, and establishes credibility through radical development transparency.

The store covers four primary views: Product Hub (Home), Action-First Editor (/create), Template Discovery, and Public Roadmap. The primary goal is a "Block Insertion" action. Secondary goal is "Save to Workspace" (Signup).

---

### Section 3 — Brand Voice & Mood

The feeling is delightfully simple, radically transparent, and maker-friendly. Like opening a fresh new page in your favorite doc-editor. The brand is high-integrity and disruptive — it focuses on "Purity over Packaging."

Copy is short, honest, and outcome-oriented. Headlines focus on "The Freedom" — "Unlimited everything for free," "The simplest way to create forms," "Privacy-friendly by design." Sentences are short. No corporate fluff. No exclamation marks.

The visual mood: "Document Purity." Pure white foundation with strictly 1px black borders. Expansive whitespace. The UI "disappears" to let the user's creation lead.

Vibe word: PRODUCTIVE.

---

### Section 4 — Core Features & Functionality

1. **"Try-Before-Join" Block Editor** — a functional, unauthenticated editor component that appears on the homepage, allowing users to insert blocks (/, +) without an account.
2. **Disruptive "Unlimited" Feature Grid** — a modular section highlighting that advanced logic, payments, and multi-page forms are free by default, established as the "Standard."
3. **Local-First Draft Persistence** — an architecture that saves unauthenticated progress to IndexedDB or LocalStorage, allowing users to return to their work before signing up.
4. **Public Transparency Hub** — a footer or sidebar component showing real-time revenue, public roadmap status, and user-requested feature counts ("Build in Public" badges).

---

### Section 5 — Design Specifications

**Visual style:** Notion-Native Minimal. Maximum whitespace. Strictly 1px borders. Zero decorative "fluff" — the product screenshots and blocks are the visuals.

**Color mode:** Light only (Document Feel).

**Color palette:**
- Background: `#FFFFFF` (Pure White — the foundation of creation)
- Surface / subtle: `#F7F7F7` (Lightest grey for labels and hover states)
- Border / Line: `#E5E7EB` (Slate-200 — ghostly separators)
- Primary text: `#000000` (Pure Black — for document authority)
- Muted text: `#737373` (Neutral-500 — for meta-data and placeholders)
- Accent (Subtle): Soft pastels (e.g., `#E0F2FE` for badges or `#FEF3C7` for highlights)
- Accent (Primary): `#000000` (Black buttons)

**Total colors: 7 values, 4 functional categories. No additional colors.**

**Typography:** Functional Geometric Sans (e.g., Inter, Public Sans) and Monospace (e.g., JetBrains Mono).
- Display / Hero heading: `clamp(32px, 5vw, 56px)`, weight 800, letter-spacing `-0.04em`, line-height 1.1 (Sans)
- Section heading (H2): `clamp(24px, 3vw, 32px)`, weight 700, letter-spacing `-0.01em`, line-height 1.2 (Sans)
- Block Text: `16px`, weight 400, letter-spacing `0`, line-height 1.6 (Sans)
- Slash Command: `14px`, weight 500, letter-spacing `0`, line-height 1.4 (Mono)
- Max line length body text: 64 characters (optimized for single-column focus)

**Spacing:** 16pt base unit.
- Section vertical spacing: `112px` desktop, `64px` mobile
- Grid gap: `24px` standard
- Container max-width: `1000px` (optimized for document layout)
- Block vertical rhythm: `12px`

**Border radius:**
- Buttons: `6px` (Modern/Sharp — not rounded, not sharp)
- Feature cards: `8px`
- Badges / chips: `4px`
- Inputs: `6px`

**Responsive:** Mobile-first creation. The block-editor remains a vertical stack. Hamburger menu focuses strictly on "Templates" and "Status." Sidebars collapse into "Drawer" overlays.

**Accessibility:** WCAG AA. High-contrast text on white. All block-insertion menus (Slash menus) must be fully navigable via keyboard (Up/Down + Enter).

**Motion:**
- Action Slide: `opacity 0 → 1`, `translateY(10px) → 0`, `300ms cubic-bezier(0.16, 1, 0.3, 1)`
- View Transition: Instant swap between "Build" and "Preview" modes.
- Hover Shift: `bg-zinc-100` transition `0.1s`.
- Forbidden: slow fades, artsy blurs, bounce easing.
- Respect: `prefers-reduced-motion: reduce` → disable all animations and transitions.

### Section 7 — Technical Specifications

- **Framework:** Next.js 14 App Router, React 18, TypeScript strict mode.
- **State:** Zustand for managing the unauthenticated `BlockTree` state.
- **Storage:** Dexie.js (IndexedDB) for local-first draft persistence.
- **Styling:** Tailwind CSS v3 with custom "Notion" spacing resets.
- **Auth:** Clerk or Auth.js for the "Workspace Transition" (Draft → Account).
- **CMS:** Contentlayer for managing public roadmap updates and template metadata.
- **Performance:** Sub-50ms keystroke-to-render latency. 100/100 Accessibility score.

```typescript
}
```

### Section 9 — User Experience

The visitor arrives seeking **Efficiency and Control**.

The first 3 seconds must answer: **"How hard is this to use?"** The block-editor demo on the home page does this work. The user must feel like they are writing a letter, not building a database.

The friction to remove: No account requirement for the first form, no "Pay to use basic logic," no complex sidebars. The UI should be invisible until a block is hovered.
