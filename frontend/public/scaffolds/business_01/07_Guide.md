# 07 — Guide
## Coaching Business Website · business_platform_01

---

### 1. Engineering Rules

- claims must remain attributable and time-bound
- form contracts must be validated server-side
- CTA hierarchy must remain consistent across pages

---

### 2. Data Rules

- normalize result data by year/program/source
- include page/funnel context in lead payload
- retain event schema versioning for analytics

---

### 3. Security Rules

- protect CRM credentials server-side
- sanitize and validate all user input
- avoid PII in logs

---

### 4. Testing Rules

- unit tests for form validation
- integration tests for lead and scholarship submissions
- analytics event assertions for core funnel steps
- regression tests for mobile form usability

---

### 5. Common Mistakes

- uncontextualized claims (“top results”) without source/year
- inconsistent CTA labels between sections
- overlong form flows that reduce completion rate
- hidden support policy links near conversion paths
